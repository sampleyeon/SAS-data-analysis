# SAS 데이터 분석 공모전

# 1. 문제 정의 (핵심 과제)

이 대회는 **이중 타겟 예측 문제**야.

### 🎯 Task 1: Churn (이탈)

* 목표: 고객 이탈 여부 확률 예측
* 출력: 0~1 확률값
* 평가: **ROC-AUC**

👉 핵심:
“누가 이탈할지”보다 **이탈할 가능성이 높은 순서 정렬**이 중요

---------------------------------------------------------------------

### 💰 Task 2: LTV (고객 생애 가치)

* 목표: 향후 1년 LTV 금액 예측
* 평가: **RMSE**

👉 핵심:
절대적인 금액 예측 정확도 중요 (회귀 문제)

-----------------------------------------------------------------------

### 🧮 최종 점수

* AUC 50% + LTV 변환 점수 50%

👉 전략적으로:

* 한쪽만 잘하면 절대 우승 못함
* **분류 + 회귀 동시에 최적화 필요**

---

# 2. 데이터 구조 (매우 중요)

총 3개 입력 테이블 + 1개 타겟

## 1) 고객 정보 (정적 데이터)

* 나이, 성별, 지역, 결혼 여부, 소득 수준 등

👉 역할:

* **기본 세그먼트 분류**
* baseline feature

---

## 2) 거래 이력 (시계열 데이터)

* 기간: 2023-07 ~ 2023-12
* 구매 금액, 채널, 카테고리, 할부 여부

👉 핵심 포인트:

* 이 데이터가 **승부처**
* 반드시 aggregation + feature engineering 필요

예:

* 최근 1개월 소비금액
* 구매 빈도
* 카테고리 다양성
* 온라인 vs 오프라인 비율
* 소비 감소 추세

---

## 3) 금융 프로필 (스냅샷)

* 신용점수
* 예적금 / 대출
* 카드 사용
* 연체일수
* 자산 변화 점수

👉 핵심:

* **이탈 + LTV 둘 다 강한 영향 변수**
* 특히:

  * 연체일수 → churn 강력 신호
  * 자산 증가 → LTV 상승 신호

---

## 4) 타겟

* churn: binary
* ltv: continuous

---

# 3. 데이터 규모

* Train: 60,000명
* Test: 40,000명

👉 특징:

* 적당한 규모 (딥러닝보다 **GBM 계열이 유리**)
* feature engineering 영향 매우 큼

---

# 4. 평가 구조의 함정 (중요)

Test가 2개로 나뉨:

* Public (2만)
* Private (2만)

👉 의미:

* **리더보드 과적합 위험 매우 큼**
* 안정적인 CV 설계 필수

---

# 5. 반드시 알아야 할 제약

### ⚠️ SAS Viya 필수 사용

* Python/R 사용 가능
* BUT:

  * 반드시 SAS Viya 활용 결과 포함해야 함

👉 전략:

* 모델링은 Python
* 일부 프로세스 or 결과를 SAS로 재현

---

# 6. 실전 전략 (분석가 관점 핵심)

## 1) Feature Engineering이 승부

특히 transaction 데이터

필수 생성:

* RFM (Recency, Frequency, Monetary)
* 소비 트렌드 (증가/감소)
* 카테고리 다양성
* 채널 비율
* 할부 비율

---

## 2) 모델 전략

### Churn

* LightGBM / XGBoost (AUC 최적화)

### LTV

* LightGBM Regressor
* 또는 log 변환 후 회귀

---

## 3) 멀티태스크 전략 (고급)

* churn 예측값을 LTV feature로 사용
* 혹은:

  * churn=1 그룹 / 0 그룹 나눠 LTV 모델 분리

---

## 4) 검증 전략 (중요)

* 단순 KFold ❌
* 추천:

  * Stratified KFold (churn 기준)
  * Group KFold (customer 기반)

---

## 5) 리더보드 전략

* 과적합 방지
* seed averaging
* ensemble 필수

---

# 7. 한 줄 요약

👉 **“고객 행동 + 금융 상태를 기반으로 이탈 확률과 미래 가치를 동시에 예측하는 고난도 고객 분석 문제”**
